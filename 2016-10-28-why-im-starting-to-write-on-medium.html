Since starting at Google almost three years ago I’ve had almost 4 different roles, each using and abusing web frameworks and tools to launch applications like Google’s internal CMS, Greentea, the new version of Google AdWords, and played a big role in spearheading early alpha version of Angular Js v2, specifically as it applied to Angular Dart (which my projects use successfully today).

Since then? I’ve joined the Dart team and work on Angular Dart, and we’ve just released 2.0.0 final, and an exciting developer preview of our Material Design components for Angular Dart — the same set of components which run our critical frontend applications (you can see it in the new AdSense frontend, for example).

Part of my goal for myself for the end of this year, and going into 2017 is to write more, publicly, and both learn and contribute with the open source community. I’ll be initially publishing a series of blogs I’m titling “Effective Frontend Engineering” based on my experiences at Google — and will tackle topics as they pertain to making frameworks versus just using them.

You can follow me on Twitter and GitHub to stay updated.
