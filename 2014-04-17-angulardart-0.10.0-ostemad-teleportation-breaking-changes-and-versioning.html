Version 0.10.0 (ostemad-teleportation) is the first new minor revision of Angular Dart and contains significant breaking changes. Summary below, please see the changelog for full details.

From now on, we'll also increment the minor revision when we introduce major breaking changes. Quick fixes and improvements that are not expected to affect existing code will increment as a patch revision. So the next set of incremental changes will be in patch release 0.10.1. The next set of breaking changes will be in 0.11.0.

Once we reach version 1.0.0 we will switch to a semantic versioning scheme as defined here.

Breaking changes in 0.10.0

  • Renaming makes type names more consistent. We're stripping out most of the Ng prefixes to make our naming follow Dart style. (We are leaving the Ng prefix on directives since directives have an ng- prefix in HTML.)
  • Filters are now called Formatters. 
  • Some changes to import directories, to reflect new naming.
Also, new in 0.9.11 (the change that convinced us we needed to start incrementing minor revisions on breaking changes)
  • You no longer need to include @MirrorsUsed in your code to limit the size of dart2js output. This is because all mirror-related code has been broken out of angular.dart as its own package application_factory.dart. In your source code, you import both packages (angular.dart and application_factory.dart). At build time, a transformer replaces application_factory.dart with application_factory_static.dart and generates all of the reflective methods statically. (See the API docs for more detail.)
  • Bootstrapping your Angular app is now performed via applicationFactory() (formerly ngBootstrap())

Major breaking changes still to come between 0.10.0 and 1.0.0:

  • Changes to how controllers are exposed into context to permit {{foo}} instead of {{cntl.foo}} syntax in applications. This will allow us to remove Controller (formerly known as NgController) which has confusing semantics. You can view issues related to this effort under the milestone Controller is Context.
  • Binding type set by the template author instead of by the component author. The component used in a template should have its type set where it is instantiated. This approach is in better alignment with web components. You can view issues related to this effort under the milestone bind-*.
